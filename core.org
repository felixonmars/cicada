* >< module

*** [note] module system

    - a module is simply a dir of files,
      with a module.jo in the dir,
      to store meta data of the module.

    - every module has a version, and the dir of a module
      should be "<module-name>/<version>/*"

    - a module will be installed to "~/.jojo"
      as "~/.jojo/module/<module-name>/<version>/*"

    - re-load -
      - problem  : loaded module can not be re-load
      - solution : to maintain a record of loaded modules

    - unique-name -
      - problem  : a name must be resolved to an unique-name
      - solution : (export) and (import)
        - in jotable, all defined name
          are unique as "module#version:name"
        - names in core can be viewed as automatically imported,
          one can use (shadow) to shadow them
